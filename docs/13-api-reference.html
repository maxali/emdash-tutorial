API reference

Every public function you can import from emdash, its sibling sub-paths, and @emdash-cms/cloudflare. Use this as a lookup when the tutorials don't cover what you need.

Where do I import from? The root "emdash" package re-exports every query helper: content, taxonomy, settings, SEO, bylines, menus, search, slug utils, i18n. The sub-paths (emdash/runtime, emdash/astro, emdash/ui, emdash/page, emdash/client, emdash/middleware) are for specific contexts — see the entry-point map below.

Entry-point map

Import pathWhen to use
emdashAll page-level queries: content, taxonomy, settings, SEO, menus, search
emdash/runtimeThe loader and DB handle — use only in live.config.ts or custom server routes
emdash/astroThe Astro integration default export + storage factories (local, s3)
emdash/uiAstro UI components: <Image />, <PortableText />, <Comments />, <WidgetArea />, etc.
emdash/pagePage metadata, JSON-LD helpers, plugin <head> hooks
emdash/clientTyped browser-side REST client (EmDashClient)
emdash/middlewareRequest-pipeline helpers for custom routes
@emdash-cms/cloudflareCloudflare adapters: D1, R2, Access, Images, Stream, Cache, sandbox
@emdash-cms/authAuth core + passkey / GitHub / Google providers

Content queries

All imported from "emdash".

getEmDashCollection(type, filter?)

Fetches all entries of a collection type with optional filter, sort, and pagination. Auto-hydrates bylines and taxonomy terms; caches per (type, filter) within a request.

import { getEmDashCollection } from 'emdash';

const { entries, error, cacheHint, nextCursor } =
  await getEmDashCollection('posts', {
    status: 'published',
    orderBy: { publishedAt: 'desc' },
    limit: 10,
  });

Signature: <T, D>(type: T, filter?: CollectionFilter) => Promise<CollectionResult<D>>

CollectionFilter: { status?, limit?, cursor?, where?, orderBy?, locale? }

CollectionResult: { entries, error, cacheHint, nextCursor }

getEmDashEntry(type, id, options?)

Fetches a single entry by ID or slug. Detects preview mode automatically; serves draft revisions when a valid preview token is present. Follows the locale fallback chain when i18n is on.

import { getEmDashEntry } from 'emdash';

const { entry, error, isPreview } = await getEmDashEntry('posts', slug);

Signature: (type, id, options?: { locale?: string }) => Promise<EntryResult>

EntryResult: { entry, error, isPreview, fallbackLocale, cacheHint }

getTranslations(type, id)

Returns every locale variant sharing the same translation group as the given entry. Use to build a language switcher.

resolveEmDashPath(path)

Matches a URL path against all registered collection URL patterns and returns the matching entry with extracted params.

getEditMeta(value)

Extracts visual-editing annotation metadata (collection, id, field) from an entry value — for integrating with visual editors.

Taxonomy / terms

All imported from "emdash".

getTaxonomyTerms(taxonomyName)

All terms for a taxonomy, hierarchical tree or flat list depending on its config.

getTerm(taxonomyName, slug)

One term with its entry count and child terms.

getEntryTerms(collection, entryId, taxonomyName?)

Terms assigned to a single entry. Pass taxonomyName to restrict to one taxonomy.

import { getEntryTerms } from 'emdash';

const tags = await getEntryTerms('posts', post.id, 'tags');

getTermsForEntries(collection, entryIds, taxonomyName)

Batch variant — eliminates N+1 queries on list pages. Returns a Map<entryId, TaxonomyTerm[]>.

getAllTermsForEntries(collection, entryIds)

All taxonomies at once. Returns Map<entryId, Record<taxonomyName, TaxonomyTerm[]>>.

getTaxonomyDefs() / getTaxonomyDef(name)

Registered taxonomy definitions (name, label, hierarchical flag, etc.).

Site settings

All imported from "emdash".

getSiteSettings()

All configured site settings, with media references resolved to full URLs. Includes site title, description, logo, favicon, and any plugin-registered keys.

import { getSiteSettings } from 'emdash';

const settings = await getSiteSettings();
// settings.title, settings.description, settings.logo, …

getSiteSetting(key)

One setting value, with request-scoped caching.

getPluginSettings(pluginId)

All persisted settings for a registered plugin.

getPluginSetting(pluginId, key)

One plugin-specific setting value.

SEO

From "emdash".

getSeoMeta(content, options?)

Resolves a complete SEO metadata object for a content entry. Falls back gracefully: explicit SEO fields → content title/excerpt → site defaults. Builds absolute canonical and OG image URLs.

import { getSeoMeta } from 'emdash';

const seo = getSeoMeta(entry, {
  siteTitle: 'My Blog',
  siteUrl: 'https://example.com',
  path: Astro.url.pathname,
});
// seo.title, seo.canonical, seo.ogImage, seo.robots …

getContentSeo(content)

Raw SEO fields from the entry without the template-fallback processing.

Bylines / authors

From "emdash".

  • getByline(id) — one byline profile.
  • getBylineBySlug(slug) — by slug.
  • getEntryBylines(collection, entryId) — credits for one entry (falls back to user-linked bylines).
  • getBylinesForEntries(collection, entryIds) — batch variant for list pages.

Navigation menus

  • getMenu(name) — fetches a nav menu by registered name; returns a resolved tree with URLs interpolated for content and taxonomy items.
  • getMenus() — lists all registered menu definitions (id, name, label).

Full-text search

  • search(query, options?) — cross-collection FTS5 search.
  • searchCollection(db, collection, query, options?) — scoped to one collection.
  • getSuggestions(db, query, options?) — autocomplete for a partial query.
  • getSearchStats(db) — indexed document counts + FTS5 health.
  • extractPlainText(portableTextBlocks) — utility for building custom indexing pipelines.

Slug / URL utils

decodeSlug(raw)

Decodes URI-encoded slug params. Use in [slug].astro before passing to getEmDashEntry.

import { decodeSlug, getEmDashEntry } from 'emdash';

const { entry } = await getEmDashEntry('posts', decodeSlug(Astro.params.slug));

slugify(text, maxLength?)

Text → URL-safe slug (lowercase, diacritics stripped, hyphens normalized). Default 80-char limit.

i18n

  • getI18nConfig() — the runtime i18n config, or null.
  • isI18nEnabled() — boolean shortcut.
  • getFallbackChain(locale) — ordered list of locales to try when content in the requested locale is absent.

I18nConfig: { defaultLocale, locales, fallback?, prefixDefaultLocale? }

Page metadata / JSON-LD

From "emdash/page".

  • getPageRuntime(locals) — retrieves the page runtime from Astro locals; undefined when emdash isn't initialized.
  • buildBlogPostingJsonLd(...) / buildWebSiteJsonLd(...) — structured-data builders.
  • safeJsonLdSerialize(data) — escaping helper for inline <script type="application/ld+json">.
  • resolvePageMetadata(...) / renderPageMetadata(...) — plugin-contributed <head> metadata pipeline.

Loader / runtime

From "emdash/runtime".

  • emdashLoader() — use inside live.config.ts with Astro's defineLiveCollection. Not for page code.
  • getDb() — the ambient Kysely DB handle; for custom server routes and plugins.

Astro integration

From "emdash/astro".

  • Default export: emdash(config: EmDashConfig) — register in astro.config.ts.
  • local(config?) — local-filesystem storage factory for EmDashConfig.storage.
  • s3(config) — S3-compatible storage factory.

Browser client

From "emdash/client" — for client-side interactive features.

  • EmDashClient — typed HTTP client for all REST ops (CRUD on content, media, taxonomies, menus, schema). Handles CSRF, auth tokens, optimistic concurrency via _rev.
  • createTransport, csrfInterceptor, tokenInterceptor, devBypassInterceptor — transport factory and interceptor helpers.
  • portableTextToMarkdown / markdownToPortableText — Portable Text ↔ Markdown converters.

Notable gaps

  • No built-in pagination component. Pagination is driven by consuming nextCursor from CollectionResult and passing it back as cursor in the next call.
  • Portable Text rendering is via the UI component only. There's no server-side "render to HTML string" utility in the public API — use <PortableText />.
  • Import from the root, not sub-paths, for queries. getEmDashCollection, getEmDashEntry, getEntryTerms, getSiteSettings are all from "emdash" — not "emdash/content". The "emdash/content" sub-path re-exports only the Portable Text / Prosemirror converters.
