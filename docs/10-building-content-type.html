Chapter 10 · Tutorial

Building for a new content type

Walk through the full loop — schema, types, list page, detail page, navigation, seed — by adding a project content type to the blog template.

What we're building. A /projects list page and /projects/:slug detail pages, powered by a new project content type defined entirely in the admin. We'll end with a typed, deployable feature.

Step 1 — Define the schema in the admin

  1. Start the dev server: pnpm dev.
  2. Open http://localhost:4321/_emdash/admin.
  3. Go to Schemas → New Schema.
  4. Name: project. Label: Projects.
  5. Add fields:
    FieldTypeRequiredNotes
    titleTextYesThe project name
    slugSlugYesAuto-derived from title
    summaryText (long)YesOne-line pitch for listing pages
    coverImageNoHero image
    bodyRich textYesPortable Text
    tagsArray of TextNoFor filtering
    urlURLNoLink to the live project
    publishedAtDateTimeYesFor sorting
  6. Save.

emdash has now created the DB tables and exposed a new project type through the API.

Step 2 — Regenerate types

In a second terminal:

npx emdash types

This writes TypeScript types for project so getEmDashCollection('project') gets full autocomplete.

Step 3 — Create a few entries

Back in the admin: Projects → New. Fill in a title, summary, upload a cover, write the body, add tags, set a date. Save as Published. Create 2–3 so your list page has something to render.

Step 4 — Build the list page

Create src/pages/projects/index.astro:

---
import BaseLayout from '../../layouts/BaseLayout.astro';
import { getEmDashCollection } from 'emdash';

const { entries: projects } = await getEmDashCollection('project', {
  status: 'published',
  orderBy: { publishedAt: 'desc' },
});
---
<BaseLayout
  title="Projects"
  description="Things we've shipped."
>
  <section class="wrap">
    <h1>Projects</h1>

    <div class="grid">
      {projects.map(p => (
        <a class="card" href={`/projects/${p.slug}`}>
          {p.data.cover && (
            <img src={p.data.cover.url} alt={p.data.title} loading="lazy" />
          )}
          <h2>{p.data.title}</h2>
          <p>{p.data.summary}</p>
          {p.data.tags && (
            <ul class="tags">
              {p.data.tags.map(t => <li>{t}</li>)}
            </ul>
          )}
        </a>
      ))}
    </div>
  </section>
</BaseLayout>

<style>

Visit /projects. You should see your cards.

Step 5 — Build the detail page

Create src/pages/projects/[slug].astro:

---
import BaseLayout from '../../layouts/BaseLayout.astro';
import { getEmDashEntry, getEmDashCollection } from 'emdash';

export async function getStaticPaths() {
  const { entries: projects } = await getEmDashCollection('project', {
    status: 'published',
  });
  return projects.map(p => ({ params: { slug: p.slug } }));
}

const { slug } = Astro.params;
const { entry: project } = await getEmDashEntry('project', slug);
if (!project) return Astro.redirect('/404');
---
<BaseLayout title={project.data.title} description={project.data.summary}>
  <article class="prose">
    {project.data.cover && (
      <img src={project.data.cover.url} alt={project.data.title} />
    )}
    <h1>{project.data.title}</h1>
    <p class="summary">{project.data.summary}</p>
    {project.data.url && (
      <p><a href={project.data.url} target="_blank" rel="noopener">Visit project →</a></p>
    )}
    <div set:html={project.renderedHtml} />
  </article>
</BaseLayout>

Step 6 — Add it to the nav

Open src/components/Header.astro:

<nav>
  <a href="/">Home</a>
  <a href="/posts">Posts</a>
  <a href="/projects">Projects</a>  {/* ← new */}
  <a href="/about">About</a>
</nav>

Step 7 (optional) — Seed it for new contributors

Add a projects array to seed/seed.json so teammates who clone the repo see sample data on first bootstrap:

{
  "posts": [ /* ... existing ... */ ],
  "projects": [
    {
      "slug": "orbit",
      "title": "Orbit",
      "summary": "A tiny scheduler for Cloudflare Cron Triggers.",
      "publishedAt": "2026-03-01T00:00:00Z",
      "status": "published",
      "tags": ["tool", "cloudflare"],
      "body": [
        { "_type": "block", "children": [ { "_type": "span", "text": "Orbit is a zero-config..." } ] }
      ]
    }
  ]
}

Step 8 — Deploy

  1. Push your code.
  2. Run pnpm deploy.
  3. Apply schema changes to production D1 (your template's remote migration script — see chapter 11).
  4. Visit the live /projects.

What you just practiced

  • Defining a schema without writing migrations.
  • Getting fully-typed content with emdash types.
  • Using getEmDashCollection with filters, sorts, and limits.
  • Building list + detail pages with Astro file routing.
  • Committing portable sample content to seed.json.

Repeat this loop for every new content type your site needs.
