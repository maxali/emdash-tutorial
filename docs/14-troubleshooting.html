Chapter 14

Troubleshooting and next steps

The common gotchas, where to go for more, and a suggested week-one learning path.

Common gotchas

SymptomLikely fix
emdash init fails locallyConfirm Node 22+ and pnpm 10+. Run corepack enable to activate pnpm.
Admin page is blank on CloudflareCheck wrangler.jsonc includes "nodejs_compat" in compatibility_flags.
D1 migrations don't apply in productionYou need --remote on any migration command, or it runs against local SQLite.
Plugins won't loadYou're probably on the free plan. Remove the worker_loaders block.
Types are stale after schema changesRun npx emdash types.
Media uploads fail in devFor Node variant, ensure ./uploads/ is writable. For Cloudflare variant, confirm MEDIA binding resolves.
"Setup wizard keeps reappearing"The admin DB was reset or the user table isn't persisted. Re-run pnpm bootstrap.

A suggested 5-day path

  1. Day 1. Scaffold the blog template with the Cloudflare variant. Run it locally. Create a post in the admin. Change a layout.
  2. Day 2. Deploy to Cloudflare. Wire up your own domain. Protect /_emdash/* with Cloudflare Access.
  3. Day 3. Define a new content type (e.g. project) in the admin. Generate types. Build a new page that queries it.
  4. Day 4. Pick a first-party plugin (forms or webhooks), enable it on the Paid plan, and wire it into a page.
  5. Day 5. Connect the MCP server to Claude Code or Cursor, and author a post entirely through an agent.

Where to get help

  • GitHub Discussions on the emdash repo — the maintainers require a Discussion before accepting feature PRs, so it's the canonical place for design questions.
  • Issues for reproducible bugs.
  • emdash@cloudflare.com for product feedback.

That's the whole surface. emdash is a small idea executed carefully: an Astro integration + a sandbox-first plugin model + the Cloudflare primitives that make scale-to-zero publishing real. Once the five files from chapter 2 live in your head, you can bend it to whatever content shape your project needs.
