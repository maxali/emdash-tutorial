Chapter 05

Getting started

Scaffold your first emdash site in under five minutes.

Prerequisites

  • Node.js 22+
  • pnpm 10+ — enable with corepack enable
  • A Cloudflare account (free tier is fine)
  • Wrangler — npm install -g wrangler && wrangler login

1. Create a project

npm create emdash@latest

You'll be asked:

  1. A template — blog, marketing, portfolio, starter, or blank.
  2. A deployment target — node (local SQLite, for self-hosting) or cloudflare (D1 + R2 + KV).

For this tutorial, pick blog + cloudflare.

2. Bootstrap

cd my-site
pnpm install
pnpm bootstrap   # runs: emdash init && emdash seed
pnpm dev

What just happened:

  • emdash init creates the database schema and migrations.
  • emdash seed loads starter content from seed/seed.json.
  • pnpm dev starts Astro's dev server (using workerd on the Cloudflare variant).

3. Open the admin

Visit http://localhost:4321/_emdash/admin.

On first visit, a setup wizard creates an admin account via passkey. If you want to skip passkey setup during local dev, use the bypass URL:

http://localhost:4321/_emdash/api/setup/dev-bypass?redirect=/_emdash/admin

4. See your site

Your public site is at http://localhost:4321. Edit a post in the admin, refresh the homepage, see the change.

What's the _emdash prefix? All emdash admin and API routes live under /_emdash/. This keeps them out of the way of your own routes — you can build a /admin page of your own without colliding.

What a first session looks like

  1. Open src/pages/index.astro and change a heading. The browser hot-reloads.
  2. Go to the admin, edit a post title, hit save. Refresh the public page.
  3. Open astro.config.mjs and see the emdash integration registered.
  4. Open src/live.config.ts and see the _emdash live collection.
  5. Browse to /_emdash/admin/schemas and look at the default content types.
