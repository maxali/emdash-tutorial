Chapter 01

What is emdash?

The open-source, TypeScript-based CMS that Cloudflare released as a successor to WordPress.

emdash is published under the MIT license and currently sits in beta preview. It was announced in early 2026 on the Cloudflare blog under the title "Introducing EmDash: a WordPress successor for the post-WordPress web."

The thesis

"EmDash is committed to building on what WordPress created: an open source publishing stack that anyone can install and use at little cost, while fixing the core problems that WordPress cannot solve."

emdash is pitched as a respectful successor rather than a hostile replacement. It inherits WordPress's open-publishing philosophy and redesigns the execution model for a serverless, AI-native world.

The four problems it targets

1. Plugin security

96% of WordPress security vulnerabilities come from plugins. In WordPress, a plugin gets unrestricted access to the database and filesystem — there's no isolation boundary. In emdash, every plugin runs in an isolated V8 sandbox and can only do what its manifest explicitly permits.

2. Marketplace lock-in

WordPress plugins are GPL-licensed (forcing derivative licensing choices) and face a 2+ week review queue with 800+ plugins waiting. emdash is MIT, and plugin authors keep their own licensing.

3. Infrastructure waste

WordPress needs always-on servers that can't scale to zero. emdash runs on Cloudflare Workers, which start in microseconds and cost nothing when idle.

4. The AI-agent future

Traditional ad-based monetization breaks when AI agents consume content rather than humans. emdash ships with a built-in MCP (Model Context Protocol) server so agents can read and write content natively — the CMS treats AI as first-class, not as a scraper to block.

Key facts at a glance

AttributeValue
LicenseMIT
LanguageTypeScript (end to end)
Frontend frameworkAstro 6
Admin UIReact 19
Rich-text formatPortable Text (structured JSON, not HTML)
DatabaseCloudflare D1 — PostgreSQL, Turso/libSQL, local SQLite also supported
Media storageCloudflare R2 — S3-compatible and local filesystem also supported
SessionsCloudflare KV (or Redis / file-based)
RuntimeCloudflare Workers (primary) or Node.js (fallback)
AuthPasskeys first, OAuth / magic links as fallbacks
StatusBeta preview (as of early 2026)
Note. The "Cloudflare-first" framing doesn't mean Cloudflare-locked. emdash ships Node.js / SQLite / local-filesystem alternatives for every Cloudflare service, so you can self-host on a VPS if you prefer. The Cloudflare path is the most ergonomic and what this tutorial focuses on.
