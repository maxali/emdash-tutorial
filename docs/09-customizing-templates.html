Chapter 09 · Tutorial

Customizing templates

Templates are not magic — they're just Astro projects. Anything you'd change in a regular Astro site, you can change here. This chapter walks through the five most common customizations.

The four places you'll edit

To change...Edit...
Site-wide colors, fonts, spacingsrc/styles/global.css
Header, footer, meta tags, wrapperssrc/layouts/BaseLayout.astro
Reusable UI pieces (cards, buttons)src/components/*.astro
Site name, default author, SEO defaultssrc/config.ts (templates vary)

Tutorial 1 — change brand colors and fonts

Templates expose design tokens as CSS custom properties. Open src/styles/global.css:

:root {
  --color-bg: #ffffff;
  --color-text: #111827;
  --color-accent: #2563eb;   /* ← change me */
  --font-sans: "Inter", system-ui, sans-serif;
  --font-serif: "Source Serif Pro", Georgia, serif;
}

[data-theme="dark"] {
  --color-bg: #0b0d10;
  --color-text: #e6edf3;
  --color-accent: #79c0ff;
}

Change --color-accent, save, and the whole site reflows in milliseconds. The accent variable is referenced from buttons, links, and focus states.

Swap the font

Three steps:

  1. Pick a font on Google Fonts or self-host from public/fonts/.
  2. Add the <link> or @font-face in BaseLayout.astro.
  3. Update --font-sans in global.css.

Tutorial 2 — edit the base layout

BaseLayout.astro is the wrapper every page uses. It owns the <html>, <head>, <Header>, <main>, and <Footer>. Here's a typical shape:

---
// src/layouts/BaseLayout.astro
import Header from '../components/Header.astro';
import Footer from '../components/Footer.astro';
import '../styles/global.css';

interface Props { title: string; description?: string; }
const { title, description } = Astro.props;
---
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>{title}</title>
    {description && <meta name="description" content={description} />}
    <link rel="icon" href="/favicon.svg" />
  </head>
  <body>
    <Header />
    <main><slot /></main>
    <Footer />
  </body>
</html>

Common edits:

  • Add OpenGraph tags — drop <meta property="og:title" ...> in the <head>.
  • Add analytics — insert a <script> tag at the end of <body>. For Cloudflare Web Analytics, it's a single line.
  • Make description required — remove the ? in the Props interface and emdash's TypeScript will flag any page that forgot to pass one.

Tutorial 3 — add or modify a component

Components live in src/components/. Here's a typed PostCard.astro you might drop in:

---
// src/components/PostCard.astro
interface Props {
  slug: string;
  title: string;
  excerpt?: string;
  publishedAt?: string;
}
const { slug, title, excerpt, publishedAt } = Astro.props;
---
<a class="post-card" href={`/posts/${slug}`}>
  <h3>{title}</h3>
  {excerpt && <p>{excerpt}</p>}
  {publishedAt && (
    <time datetime={publishedAt}>
      {new Date(publishedAt).toLocaleDateString()}
    </time>
  )}
</a>

<style>
  .post-card {
    display: block;
    padding: 1rem 1.25rem;
    border: 1px solid var(--color-border);
    border-radius: 8px;
    color: var(--color-text);
    text-decoration: none;
  }
  .post-card:hover { border-color: var(--color-accent); }
</style>

Use it anywhere:

---
import PostCard from '../components/PostCard.astro';
import { getEmDashCollection } from 'emdash';

const { entries: posts } = await getEmDashCollection('post');
---
{posts.map(p => <PostCard {...p.data} slug={p.slug} />)}

Tutorial 4 — add an interactive React island

Every template installs @astrojs/react already. Drop a React component in and hydrate it with client:*:

// src/components/Counter.tsx
import { useState } from 'react';

export default function Counter() {
  const [n, setN] = useState(0);
  return <button onClick={() => setN(n + 1)}>Clicked {n}</button>;
}
// In any .astro page
---
import Counter from '../components/Counter.tsx';
---
<Counter client:load />

Everything else on the page remains zero-JS; only the Counter ships React.

Tutorial 5 — update seed content

seed/seed.json is the starter content loaded by emdash seed. Edit it to change what a fresh clone starts with:

{
  "posts": [
    {
      "slug": "welcome",
      "title": "Welcome to our blog",
      "publishedAt": "2026-04-20T10:00:00Z",
      "status": "published",
      "body": [ /* Portable Text blocks */ ]
    }
  ],
  "categories": [ { "slug": "general", "name": "General" } ]
}

After editing, reseed:

pnpm emdash seed
Seed is not a sync. Editing seed.json does not push those changes into an already-populated database. Behavior on re-seed depends on the template: some upsert, some require --reset. Check your template's script.

Where not to customize

Don't edit files under node_modules/emdash/ or node_modules/@emdash-cms/. Those are the CMS itself. If you need to change admin behavior, write a plugin. If you need to change the admin UI's look, the team is working on theming hooks — track the GitHub repo for updates.
