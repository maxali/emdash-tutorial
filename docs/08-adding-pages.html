Chapter 08 · Tutorial

Adding pages

Three common cases: a static page, a dynamic page driven by content, and a page that shows a filtered list. All three follow the same Astro routing rules.

The one rule of Astro routing

Every file in src/pages/ becomes a route. Nothing else is a route. File name → URL mapping:

FileURL
src/pages/index.astro/
src/pages/about.astro/about
src/pages/blog/index.astro/blog
src/pages/posts/[slug].astro/posts/:slug (dynamic)
src/pages/rss.xml.ts/rss.xml (endpoint)

Tutorial 1 — add a static "About" page

We'll use the BaseLayout that ships with every template so the new page picks up the header, footer, and global styles for free.

Create the file

// src/pages/about.astro
---
import BaseLayout from '../layouts/BaseLayout.astro';
---
<BaseLayout title="About us" description="Who we are and what we do.">
  <section class="prose">
    <h1>About us</h1>
    <p>We build tools for the web.</p>
  </section>
</BaseLayout>

Verify

With pnpm dev running, open http://localhost:4321/about. The page renders; HMR means no restart needed.

Link to it from the nav

Open src/components/Header.astro (or whichever file owns your nav) and add a link:

<nav>
  <a href="/">Home</a>
  <a href="/about">About</a>  {/* ← new */}
</nav>

Tutorial 2 — add a dynamic page from a content type

Your admin has a post content type. We'll build /posts/:slug pages if the template doesn't have them, or study how the template already does it.

Create the dynamic route

// src/pages/posts/[slug].astro
---
import BaseLayout from '../../layouts/BaseLayout.astro';
import { getEmDashEntry, getEmDashCollection } from 'emdash';

export async function getStaticPaths() {
  const { entries: posts } = await getEmDashCollection('post');
  return posts.map(p => ({ params: { slug: p.slug } }));
}

const { slug } = Astro.params;
const { entry: post } = await getEmDashEntry('post', slug);
if (!post) return Astro.redirect('/404');
---
<BaseLayout title={post.data.title}>
  <article class="prose">
    <h1>{post.data.title}</h1>
    <time>{post.data.publishedAt}</time>
    <div set:html={post.renderedHtml} />
  </article>
</BaseLayout>
Static paths vs SSR. getStaticPaths is for pre-rendering at build time. Because emdash templates use output: 'server', you can also skip getStaticPaths entirely — the route will be resolved at request time from Astro.params. Use getStaticPaths for speed-critical, rarely-changing content; use pure SSR when content changes frequently.

Try it

Create a post in the admin (or use the seeded ones). Visit /posts/<slug> — your new page renders.

Tutorial 3 — add a filtered index page

Build a page that only shows published posts from the last 30 days.

// src/pages/recent.astro
---
import BaseLayout from '../layouts/BaseLayout.astro';
import { getEmDashCollection } from 'emdash';

const thirtyDaysAgo = new Date();
thirtyDaysAgo.setDate(thirtyDaysAgo.getDate() - 30);

const { entries: recent } = await getEmDashCollection('post', {
  status: 'published',
  where: { publishedAt: { gte: thirtyDaysAgo.toISOString() } },
  orderBy: { publishedAt: 'desc' },
  limit: 20,
});
---
<BaseLayout title="Recent posts">
  <h1>Recent posts</h1>
  <ul>
    {recent.map(p => (
      <li>
        <a href={`/posts/${p.slug}`}>{p.data.title}</a>
        <time>{new Date(p.data.publishedAt).toLocaleDateString()}</time>
      </li>
    ))}
  </ul>
</BaseLayout>

Endpoints: JSON, RSS, sitemaps

Any .ts or .js file in src/pages/ becomes an HTTP endpoint. Great for RSS feeds, JSON APIs, and sitemaps.

// src/pages/posts.json.ts
import type { APIRoute } from 'astro';
import { getEmDashCollection } from 'emdash';

export const GET: APIRoute = async () => {
  const { entries: posts } = await getEmDashCollection('post', {
    status: 'published',
  });

  return new Response(
    JSON.stringify(posts.map(p => ({
      slug: p.slug,
      title: p.data.title,
      publishedAt: p.data.publishedAt,
    }))),
    { headers: { 'Content-Type': 'application/json' } }
  );
};

Now GET /posts.json returns the list.
