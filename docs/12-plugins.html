Chapter 12

Plugins and extensibility

emdash's flagship architectural choice — sandboxed plugins — and how the MCP server turns the CMS into an AI-native authoring surface.

The sandbox model

Every plugin runs inside its own V8 isolate (a Dynamic Worker). A plugin declares a capability manifest up front:

{
  "name": "my-notifier",
  "capabilities": [
    "read:content.post",
    "network:send",
    "email:send"
  ]
}

The runtime enforces those capabilities. A plugin that didn't declare write:content.page cannot write to pages — not in a sneaky way, not ever. Contrast with WordPress, where any plugin can DROP TABLE users the moment it's activated.

Why V8 isolates? They're the same sandbox unit Chrome uses per browser tab. Cheap (microseconds to start), deeply battle-tested, and inherently untrusted by default. That's the right default for third-party code.

First-party plugins

A few ship as official packages:

PluginWhat it does
@emdash-cms/plugin-formsForm submissions (included with the blog-cloudflare template).
@emdash-cms/plugin-webhook-notifierFire webhooks on content events.
@emdash-cms/plugin-audit-logTrack admin actions for compliance.

Registering a plugin

Plugins are registered in astro.config.mjs, in the plugins array passed to the emdash integration:

import emdash from 'emdash/astro';
import forms from '@emdash-cms/plugin-forms';

export default defineConfig({
  integrations: [
    emdash({
      adapter: cloudflareAdapter({ /* ... */ }),
      plugins: [
        forms({
          submissionsPerMinute: 5,
          recipients: ['hello@example.com'],
        }),
      ],
    }),
  ],
});

Free-tier caveat

Plugins require the Workers Paid plan. Cloudflare's Dynamic Workers / Worker Loader (the sandbox primitive) is only available on paid accounts ($5/month minimum). On the free plan, comment out the worker_loaders block in wrangler.jsonc and skip plugins. Everything else — admin, content, media, auth — still works.

The MCP server — emdash's AI story

emdash exposes an MCP (Model Context Protocol) server, which means Claude Code, Cursor, and any MCP-compatible agent can read and write content directly — create posts, update schemas, upload media — through authenticated tool calls.

This is the "AI-native" angle the Cloudflare team emphasized at launch: your CMS talks the same protocol your coding agents do. An agent can:

  • Scaffold new content types based on a natural-language spec.
  • Draft and publish posts with the same permission boundaries as a human editor.
  • Bulk-edit content safely, scoped by role.

Writing your own plugin

A plugin is an npm package that exports a default function. The function receives a capability-scoped API and returns hook handlers:

// hello-plugin/src/index.ts
import { definePlugin } from '@emdash-cms/plugin-sdk';

export default definePlugin({
  name: 'hello',
  capabilities: ['read:content.post', 'log:write'],

  hooks: {
    'content.published': async (evt, api) => {
      api.log.info(`Published: ${evt.entry.title}`);
    },
  },
});

Because the plugin runs in an isolate, api is the only way to touch the outside world. There is no fs, no arbitrary fetch, no DB client — just what the host chose to grant.
